digi2 Essentials v1.0

Component library for Webflow. One script tag, modular, on-demand loading.

Installation

Single script tag in Webflow → Site Settings → Custom Code → Head Code:

<!-- Load only what you need via d2-* attributes -->
<script
  src="https://cdn.jsdelivr.net/gh/Digi2-Agency/digi2-essentials@latest/dist/digi2-loader.min.js"
  d2-gtm="GTM-XXXXXXX"
  d2-popups
  d2-cookies
  d2-forms
></script>
AttributeModuleSize (min)
d2-gtm="ID"Consent Mode V2 + GTM2.5 KB
d2-popupsPopup / modal manager8.4 KB
d2-cookiesCookie helpers1.1 KB
d2-formsForms + validation + UTM12.1 KB

Loader itself: 3.4 KB. Only requested modules are fetched.

<script src="...digi2-loader.min.js" d2-popups d2-debug-mode></script>

Custom Error Messages

Override default error messages by passing errorMessages when creating a form. Use {param} as a placeholder for the rule's parameter value.

digi2.forms.create('contact', {
  errorMessages: {
    required:  'Please fill this in.',
    minLength: 'Too short — at least {param} characters.',
    maxLength: 'Too long — max {param} characters.',
    email:     'That doesn\'t look like an email.',
    phone:     'Enter a valid phone number.',
  }
})

Built-in defaults are provided for every rule. Your overrides only replace the ones you specify.

Error Elements

Setup

Add one d2-form-error-text element per field, hidden by default. The module finds the closest one to each input and sets its text from errorMessages.

<label>
  Email
  <input type="email" name="EMAIL" />
  <div d2-form-error-text style="display:none"></div>
</label>

When validation fails, the first error's message is injected as text content and display: flex is applied. When valid, it's hidden and cleared.

Error Display Modes

// Per-field (default)
{ errorDisplay: 'inline' }

// Summary block above submit
{ errorDisplay: 'summary', summaryMessage: 'Fix these:' }

Custom Input Styles on Error

{
  inputOnError: { borderColor: '#ef4444', boxShadow: '0 0 0 2px rgba(239,68,68,0.2)' },
  inputOnValid: { borderColor: '', boxShadow: '' },
}

Auto Error Elements

Set autoErrorElements: true to auto-create [d2-form-error-text] elements — no manual Webflow setup needed. Each validated input gets wrapped in a <label> (if not already inside one) and an absolutely positioned error text <div> is appended below.

{
  autoErrorElements: true,
  errorFontSize:     '11px',          // font-size for error text
  errorColor:        '#ef4444',       // color for error text
  errorLocation:     'below',        // 'below' | 'above'
  errorWrapClass:    'my-field',     // CSS class on <label> wrapper
  errorWrapStyle:    { marginBottom: '16px' },  // CSS object for <label> wrapper
  errorTextStyle:    { padding: '2px 0' },    // CSS object for error text (overrides defaults)
}

Skips checkboxes, radios, and hidden inputs. If an input already has a manual [d2-form-error-text] element nearby, it is reused and no wrapper is created.

Events

digi2.on('loaded', fn)              // all modules loaded
digi2.on('module:loaded', fn)       // single module (receives name)
digi2.on('module:error', fn)        // module failed (receives name)
digi2.on('consent:updated', fn)     // consent changed (receives state)
digi2.off('loaded', fn)             // remove listener
digi2.off('loaded')                // remove all for event
digi2.emit('custom', data)         // emit custom event
digi2.onReady(fn)                 // alias for on('loaded')

Subscribing to loaded after modules already loaded calls fn immediately.
